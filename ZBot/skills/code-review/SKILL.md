# 代码审查助手

专业的代码审查技能，帮助发现代码问题并提供改进建议。

## 审查维度

### 1. 代码质量
- 命名规范（变量、函数、类）
- 代码结构和组织
- 注释和文档完整性
- 代码重复度

### 2. 潜在问题
- 空指针/未定义引用
- 资源泄漏（文件、连接、内存）
- 异常处理缺失
- 边界条件处理

### 3. 安全隐患
- SQL 注入风险
- XSS 跨站脚本
- 敏感信息硬编码
- 不安全的依赖

### 4. 性能问题
- 不必要的循环
- N+1 查询问题
- 内存占用过大
- 低效算法

### 5. 最佳实践
- 设计模式应用
- SOLID 原则
- DRY 原则
- 错误处理策略

## 使用方式

### 审查单个文件
```
审查 src/utils/helper.py 的代码质量
```

### 审查特定问题
```
检查这段代码有没有安全问题：[代码片段]
这个函数的性能可以优化吗？
```

### 审查整个目录
```
审查 src/ 目录下的所有 Python 文件
```

### 对比审查
```
对比审查 main.py 和 refactor_main.py
```

## 输出格式

审查报告包含：

1. **问题等级**
   - 🔴 严重：必须修复
   - 🟡 警告：建议修复
   - 🔵 建议：可选优化

2. **问题描述**
   - 具体位置（文件:行号）
   - 问题说明
   - 修复建议

3. **代码示例**
   - 问题代码
   - 推荐写法

## 支持的语言

Python、JavaScript、TypeScript、Go、Java、Rust、C/C++、SQL 等

## 注意事项

- 审查结果仅供参考，最终决策由开发者做出
- 某些警告可能是误报，需要结合上下文判断
- 建议优先处理严重和警告级别的问题
