# Nanobot 项目架构分析

## 一、项目定位

**Nanobot 是一个多通道AI智能体框架**，核心能力：
- 支持多LLM提供商（OpenAI/Anthropic/国内厂商）统一调用
- 支持多通道通信（CLI/WhatsApp/Telegram等）
- 内置工具系统（文件操作/Shell执行/定时任务/MCP协议）
- 长期记忆与会话管理

| `context.py` | 上下文构建器 | `ContextBuilder` |
| `memory.py` | 长期记忆存储 | `MemoryStore` |
| `skills.py` | 技能加载器 | `SkillsLoader` |
| `subagent.py` | 子代理管理 | `SubAgent` |

## 四、核心设计模式

### 1. 抽象工厂模式
- `Tool` 基类 → 多种具体工具实现
- `BaseChannel` → 多种通道实现
- `LLMProvider` → LiteLLMProvider

### 2. 注册表模式
- `PROVIDERS` 注册表管理所有LLM提供商规格
- `ToolRegistry` 动态注册工具
- `discover_all()` 自动发现所有通道

### 3. 发布-订阅模式
- `MessageBus` 实现模块间解耦通信
- 入站消息/出站消息分离处理

### 4. 策略模式
- 不同LLM厂商配置通过 `ProviderSpec` 策略对象驱动
- 无硬编码if-elif，完全配置化
