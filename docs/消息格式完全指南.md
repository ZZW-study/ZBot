## 1. 消息流程概览

```
用户输入
    ↓
┌─────────────────────────────────────────────────────────────┐
│  ContextBuilder.build_messages()                            │
│  ├── System Prompt（身份 + 引导文件 + 记忆 + 技能）          │
│  ├── History（历史对话）                                     │
│  └── User Message（运行时上下文 + 用户内容 + 媒体附件）       │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│  LiteLLMProvider.chat()                                     │
│  ├── 消息清洗（空内容处理、字段过滤）                         │
│  ├── 缓存控制注入（Anthropic/OpenRouter）                    │
│  └── 工具定义（tools 参数）                                  │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│  大模型 API                                                  │
│  返回 LLMResponse（content + tool_calls + usage）           │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│  AgentLoop._run_agent_loop()                                │
│  ├── 有工具调用 → 执行工具 → 写回 tool 结果 → 循环           │
│  └── 无工具调用 → 返回最终回复                               │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│  SessionManager.save()                                      │
│  持久化到 JSONL 文件                                         │
└─────────────────────────────────────────────────────────────┘
```

## 2. 发送给大模型的消息格式

### 2.1 消息字段总表

发送给大模型的 `messages` 参数是一个列表，每条消息包含以下字段：

| 字段                  | 类型           | 必需 | 适用角色                | 说明                                                       |
| --------------------- | -------------- | :--: | ----------------------- | ---------------------------------------------------------- |
| `role`              | string         |  ✅  | 所有                    | 消息角色：`system` / `user` / `assistant` / `tool` |
| `content`           | string\| array |  ✅  | system, user, assistant | 消息内容。纯文本为 string，多模态为 array                  |
| `tool_calls`        | array          |  ❌  | assistant               | 工具调用列表，格式见下表                                   |
| `tool_call_id`      | string         |  ✅  | tool                    | 关联的 assistant 工具调用 ID                               |
| `name`              | string         |  ❌  | tool                    | 工具名称（发送前会被清洗掉，仅用于内部日志）               |
| `reasoning_content` | string         |  ❌  | assistant               | 推理内容（部分模型支持）                                   |

#### `tool_calls` 数组元素结构

| 字段                   | 类型   | 说明                            |
| ---------------------- | ------ | ------------------------------- |
| `id`                 | string | 工具调用唯一标识（9位字母数字） |
| `type`               | string | 固定值 `"function"`           |
| `function.name`      | string | 工具名称                        |
| `function.arguments` | string | 参数 JSON 字符串                |

#### `content` 多模态数组元素结构

| 字段              | 类型   | 说明                               |
| ----------------- | ------ | ---------------------------------- |
| `type`          | string | `"text"` 或 `"image_url"`      |
| `text`          | string | 文本内容（type=text 时）           |
| `image_url.url` | string | 图片 data URL（type=image_url 时） |

### 2.2 工具定义字段总表（tools 参数）

| 字段                     | 类型   | 说明                                |
| ------------------------ | ------ | ----------------------------------- |
| `type`                 | string | 固定值 `"function"`               |
| `function.name`        | string | 工具名称                            |
| `function.description` | string | 工具描述                            |
| `function.parameters`  | object | JSON Schema 格式的参数定义          |
| `cache_control`        | object | 缓存控制（仅 Anthropic/OpenRouter） |

### 2.3 完整请求示例

```json
{
  "model": "claude-sonnet-4-6-20250929",
  "messages": [
    {
      "role": "system",
      "content": "# ZBot\n你是 ZBot，一名可靠、直接、善于执行的 AI 助手..."
    },
    {
      "role": "user",
      "content": "[运行时上下文]\n当前时间：2026-04-14 15:30\n\n帮我查一下北京今天的天气"
    },
    {
      "role": "assistant",
      "content": "让我查询一下北京的天气。",
      "tool_calls": [
        {
          "id": "abc123def",
          "type": "function",
          "function": {
            "name": "exec",
            "arguments": "{\"command\": \"curl -s \\\"wttr.in/Beijing?format=3\\\"\"}"
          }
        }
      ]
    },
    {
      "role": "tool",
      "tool_call_id": "abc123def",
      "name": "exec",
      "content": "Beijing: ⛅️ +18°C"
    }
  ],
  "tools": [
    {
      "type": "function",
      "function": {
        "name": "exec",
        "description": "执行 shell 命令",
        "parameters": {
          "type": "object",
          "properties": {
            "command": {"type": "string", "description": "要执行的命令"}
          },
          "required": ["command"]
        }
      }
    }
  ],
  "tool_choice": "auto",
  "max_tokens": 4096,
  "temperature": 0.1
}
```

## 3. 大模型返回的消息格式

### 3.1 返回字段总表

大模型返回的消息被解析为 `LLMResponse` 对象，包含以下字段：

| 字段                  | 类型                  | 说明                                                                   |
| --------------------- | --------------------- | ---------------------------------------------------------------------- |
| `content`           | string\| None         | 回复文本内容                                                           |
| `tool_calls`        | list[ToolCallRequest] | 工具调用列表，空列表表示无工具调用                                     |
| `finish_reason`     | string                | 结束原因：`stop` / `tool_calls` / `error`                        |
| `usage`             | dict                  | Token 统计：`prompt_tokens`, `completion_tokens`, `total_tokens` |
| `reasoning_content` | string\| None         | 推理内容（部分模型支持，如 DeepSeek）                                  |
| `has_tool_calls`    | bool                  | 属性，判断 `tool_calls` 是否非空                                       |

#### `tool_calls` 列表元素结构（ToolCallRequest）

| 字段          | 类型   | 说明                                    |
| ------------- | ------ | --------------------------------------- |
| `id`        | string | 工具调用唯一标识（标准化为9位字母数字） |
| `name`      | string | 工具名称                                |
| `arguments` | dict   | 工具参数（已解析为字典）                |

### 3.2 原始 API 响应示例

**纯文本回复：**

```json
{
  "choices": [{
    "message": {
      "role": "assistant",
      "content": "你好！我是 ZBot，有什么可以帮助你的吗？"
    },
    "finish_reason": "stop"
  }],
  "usage": {
    "prompt_tokens": 150,
    "completion_tokens": 20,
    "total_tokens": 170
  }
}
```

**包含工具调用：**

```json
{
  "choices": [{
    "message": {
      "role": "assistant",
      "content": "让我帮你查询天气。",
      "tool_calls": [{
        "id": "call_abc123xyz",
        "type": "function",
        "function": {
          "name": "exec",
          "arguments": "{\"command\": \"curl -s \\\"wttr.in/Beijing?format=3\\\"\"}"
        }
      }]
    },
    "finish_reason": "tool_calls"
  }],
  "usage": {
    "prompt_tokens": 200,
    "completion_tokens": 50,
    "total_tokens": 250
  }
}
```

### 3.3 解析后的 LLMResponse 示例

```python
# 纯文本回复
LLMResponse(
    content="你好！我是 ZBot，有什么可以帮助你的吗？",
    tool_calls=[],
    finish_reason="stop",
    usage={"prompt_tokens": 150, "completion_tokens": 20, "total_tokens": 170},
    reasoning_content=None
)

# 包含工具调用
LLMResponse(
    content="让我帮你查询天气。",
    tool_calls=[
        ToolCallRequest(
            id="abc123def",
            name="exec",
            arguments={"command": "curl -s \"wttr.in/Beijing?format=3\""}
        )
    ],
    finish_reason="tool_calls",
    usage={"prompt_tokens": 200, "completion_tokens": 50, "total_tokens": 250},
    reasoning_content=None
)

# 包含推理内容
LLMResponse(
    content="根据天气查询结果，北京今天天气晴朗。",
    tool_calls=[],
    finish_reason="stop",
    usage={"prompt_tokens": 300, "completion_tokens": 80, "total_tokens": 380},
    reasoning_content="用户想了解北京天气，我已经通过 exec 工具获取了天气信息..."
)
```

## 4. 解析后的消息格式

### 4.1 消息清洗处理

`LiteLLMProvider._sanitize_messages()` 方法对消息进行以下处理：

1. **空内容处理**：空字符串替换为 `"(空内容)"`
2. **字段过滤**：只保留标准字段（role, content, tool_calls, tool_call_id, name, reasoning_content）
3. **工具ID标准化**：非9位字母数字ID转为哈希截取
4. **移除 tool 消息中的 name 字段**（OpenAI 兼容）

### 4.2 缓存控制注入（Anthropic/OpenRouter）

对于支持提示词缓存的模型，会注入 `cache_control` 标记：

```json
{
  "role": "system",
  "content": [
    {
      "type": "text",
      "text": "# ZBot\n你是 ZBot...",
      "cache_control": {"type": "ephemeral"}
    }
  ]
}
```

工具定义也会注入缓存标记：

```json
[
  {"type": "function", "function": {"name": "read_file", ...}},
  {"type": "function", "function": {"name": "exec", ...}, "cache_control": {"type": "ephemeral"}}
]
```

### 示例 3：读取文件并编辑（多次工具调用）

**用户输入：** "读取 config.json 文件，把里面的 debug 改成 false"

```python
LLMResponse(
    content="我来帮你读取并修改配置文件。",
    tool_calls=[
        ToolCallRequest(
            id="read001",
            name="read_file",
            arguments={"path": "config.json"}
        )
    ],
    finish_reason="tool_calls"
)
```

**工具执行结果：**

```json
{"debug": true, "port": 8080, "host": "localhost"}
```

**第二轮响应：**

```python
LLMResponse(
    content="我看到配置文件中 debug 当前是 true，我来把它改成 false。",
    tool_calls=[

            id="edit001",
            name="edit_file",
            arguments={
                "path": "config.json",
                "old_text": "\"debug\": true",
                "new_text": "\"debug\": false"
            }
        )
    ],
    finish_reason="tool_calls"
)
```

**第三轮响应（最终回复）：**

```python
LLMResponse(
    content="已完成修改！config.json 文件中的 debug 已从 true 改为 false。",
    tool_calls=[],
    finish_reason="stop"
)
```

## 6. 会话持久化格式

### 6.1 JSONL 文件格式

会话保存在 `workspace/sessions/` 目录下，每个会话一个 `.jsonl` 文件：

```
sessions/
  cli_default.jsonl
  telegram_8281248569.jsonl
```

### 6.2 文件内容示例

**cli_default.jsonl：**

```json
{"_type": "metadata", "key": "cli:default", "created_at": "2026-04-14T10:00:00", "updated_at": "2026-04-14T15:30:00", "metadata": {}, "last_consolidated": 0}
{"role": "user", "content": "你好，介绍一下你自己", "timestamp": "2026-04-14T10:00:05"}
{"role": "assistant", "content": "你好！我是 ZBot 🐈，你的专属 AI 助手...", "timestamp": "2026-04-14T10:00:10"}
{"role": "user", "content": "帮我查一下北京今天的天气", "timestamp": "2026-04-14T15:30:00"}
{"role": "assistant", "content": "让我查询一下北京的天气。", "tool_calls": [{"id": "abc123def", "type": "function", "function": {"name": "exec", "arguments": "{\"command\": \"curl -s \\\"wttr.in/Beijing?format=3\\\"\"}"}}], "timestamp": "2026-04-14T15:30:05", "tools_used": ["exec"]}
{"role": "tool", "tool_call_id": "abc123def", "name": "exec", "content": "Beijing: ⛅️ +18°C", "timestamp": "2026-04-14T15:30:06"}
{"role": "assistant", "content": "北京今天的天气：⛅️ 多云，气温 +18°C。", "timestamp": "2026-04-14T15:30:10"}
```

### 6.3 特殊字段说明

| 字段                  | 说明                                          |
| --------------------- | --------------------------------------------- |
| `_type: "metadata"` | 元数据行标记                                  |
| `last_consolidated` | 已归档的消息索引                              |
| `tools_used`        | 本轮使用的工具列表（挂载在 assistant 消息上） |
| `timestamp`         | 消息时间戳                                    |

### 6.4 工具结果截断

当工具返回结果超过 2000 字符时，会自动截断：

```json
{"role": "tool", "tool_call_id": "xyz789", "name": "read_file", "content": "很长的内容...（前2000字符）\n……（内容已截断）", "timestamp": "..."}
```
