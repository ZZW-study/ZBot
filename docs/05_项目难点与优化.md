# Nanobot 项目难点、解决方案与优化方向

## 一、核心技术难点

### 难点1：多LLM厂商API兼容

#### 问题描述
不同厂商的API存在大量差异：
- **模型命名**：`gpt-4o` vs `claude-3-opus` vs `deepseek-chat`
- **认证方式**：API Key vs OAuth vs 本地部署
- **参数支持**：部分厂商不支持temperature、reasoning_effort
- **响应格式**：工具调用、思考内容字段名不一致

#### 解决方案：注册表驱动设计

```python
# nanobot/providers/registry.py

PROVIDERS = [
    ProviderSpec(
        name="anthropic",
        label="Anthropic",
        litellm_prefix="anthropic",
        env_key="ANTHROPIC_API_KEY",
        supports_prompt_caching=True,  # 支持提示词缓存
        model_overrides=[
            ("claude-3-5-haiku", {"temperature": 1.0}),  # 强制温度
        ],
    ),
    ProviderSpec(
        name="openai",
        label="OpenAI",
        litellm_prefix="openai",
        env_key="OPENAI_API_KEY",
        skip_prefixes=["o1", "o3"],  # 这些模型不加前缀
    ),
    ProviderSpec(
        name="openrouter",
        label="OpenRouter",
        litellm_prefix="openrouter",
        env_key="OPENROUTER_API_KEY",
        is_gateway=True,  # 网关模式
        supports_prompt_caching=True,
    ),
]
```

**优势**：
- 新增厂商只需加配置，无需改代码
- 特殊逻辑通过策略对象驱动
- 100%避免if-elif硬编码

### 难点2：工具调用迭代控制

#### 问题描述
LLM可能：
- 连续调用多个工具
- 工具返回后再次调用工具
- 陷入死循环（工具A调用工具B，工具B又调用工具A）

#### 解决方案：迭代计数+消息累积

```python
# nanobot/agent/loop.py

async def process_direct(self, message: str, ...) -> str:
    messages = [...]
    iterations = 0
    
    while iterations < self.max_iterations:  # 防止死循环
        response = await self.provider.chat(messages=messages, tools=tools)
        
        # 无工具调用，返回结果
        if not response.tool_calls:
            break
        
        # 执行工具
        iterations += 1
        for tc in response.tool_calls:
            result = await self._execute_tool(tc)
            # 关键：工具结果加入消息历史
            messages.append({
                "role": "tool",
                "tool_call_id": tc.id,
                "content": result,
            })
    
    return response.content
```

**关键点**：
- `max_iterations`默认10次，可配置
- 每次工具结果都加入上下文，LLM能看到执行历史
- LLM自主决定是否继续调用工具

### 难点3：上下文窗口溢出

#### 问题描述
对话越来越长，最终超出LLM上下文限制（如Claude的200K token）。

#### 解决方案：三层策略

**策略1：记忆窗口截断**
```python
# nanobot/session/manager.py

def get_history(self, max_messages: int = 500) -> list[dict]:
    """只保留最近N条消息"""
    return self.messages[-max_messages:]
```

**策略2：消息合并总结**
```python
# nanobot/agent/memory.py

def consolidate(self, session: Session) -> None:
    """AI总结历史对话，写入MEMORY.md"""
    summary = await self.agent.process_direct(
        f"Summarize the following conversation:\n{session.messages}"
    )
    self.memory_file.write_text(summary)
    session.last_consolidated = len(session.messages)
```

**策略3：渐进式技能加载**
```python
# nanobot/agent/skills.py

def build_skills_summary(self) -> str:
    """先生成XML摘要，需要时再读取完整技能"""
    return """
    <skills>
      <skill available="true">
        <name>web-search</name>
        <description>Search the web with Brave API</description>
        <location>~/.nanobot/skills/web-search/SKILL.md</location>
      </skill>
    </skills>
    """
```

### 难点4：Shell命令安全执行

#### 问题描述
AI可能执行危险命令：`rm -rf /`、`shutdown`、访问系统敏感文件。

#### 解决方案：三层防护

**防护1：危险命令黑名单**
```python
# nanobot/agent/tools/shell.py

DENY_PATTERNS = [
    r"\brm\s+-[rf]{1,2}\b",          # rm -rf
    r"\b(shutdown|reboot|poweroff)\b",  # 关机命令
    r":\(\)\s*\{.*\};\s*:",          # fork bomb
]
```

**防护2：白名单机制**
```python
if self.allow_patterns:
    if not any(re.search(p, command) for p in self.allow_patterns):
        return "Error: Command blocked (not in allowlist)"
```

**防护3：路径越界检查**
```python
if self.restrict_to_workspace:
    # 检查命令中的所有绝对路径
    for path in self._extract_absolute_paths(command):
        if not path.startswith(self.workspace):
            return "Error: Path outside workspace"
```

## 二、性能优化方向

### 优化1：提示词缓存（已实现）

**问题**：多轮对话重复发送相同系统提示，浪费Token。

**方案**：为Anthropic/OpenRouter自动注入缓存控制。
```python
def _apply_cache_control(self, messages, tools):
    """为系统消息添加缓存标记"""
    for msg in messages:
        if msg["role"] == "system":
            msg["content"] = [{
                "type": "text",
                "text": msg["content"],
                "cache_control": {"type": "ephemeral"}  # 缓存标记
            }]
    return messages, tools
```

**效果**：系统提示只计费一次，后续对话Token费用降低50%+。

for tc in response.tool_calls:

# 分析工具依赖关系
independent, dependent = analyze_dependencies(response.tool_calls)

results = await asyncio.gather(*[
    self._execute_tool(tc) for tc in independent

async def warmup_cache(self):
    """启动时加载最近使用的会话到内存"""

    for session in recent_sessions:
        self.session_manager.get_or_create(session["key"])
